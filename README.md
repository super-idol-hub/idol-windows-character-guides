# 偶像 Windows 角色开发规范库

面向粉丝非商业项目的 Windows 桌面互动角色开发流程。仓库把“小曦薇”项目中验证过的素材、动作、动画、皮肤、QA、独立 EXE 和发布经验整理成两套可复用规范。

这是一个**流程文档仓库**，不是程序、自动生成器、素材库或 EXE 下载站。

## 两套规范

- [女性偶像 Windows 角色生成与开发规范](docs/女性偶像Windows角色生成与开发规范.md)：以“小曦薇”v3.0.4 为真实案例，记录从参考素材到单文件 EXE 的完整过程和故障修复经验。
- [男性偶像 Windows 角色生成与开发规范](docs/男性偶像Windows角色生成与开发规范.md)：复用同一套 24 行、176 帧运行时协议，提供男性角色默认动作组、气质配置和专项视觉 QA。

| 文档 | 状态 | 本仓库是否包含人物素材/EXE |
|---|---|---|
| 女性版 | 基于已经完成的“小曦薇”v3.0.4 外部实作案例 | 否 |
| 男性版 | 由同一协议派生的设计规范，尚未针对具体男艺人完成首次实作 | 否 |

两套规范的技术底座相同：

- 写实或半写实全身透明人物。
- `528×808` RGBA 高清关键帧。
- 24 行、176 个归档帧。
- 普通动作至少 24 个运行时显示阶段。
- 双向运动网格与单一人物轮廓补间，避免双人重影。
- 16 方向近距离鼠标视线。
- 持久坐姿、睡眠、皮肤接口、换装动画和自动/人工 QA。
- Windows 独立 EXE；可选皮肤放在 EXE 同级 `skins` 文件夹。

性别版本只是默认动作起点，不是行为限制。实际动作应根据艺人的公开气质、舞台习惯、影视角色、服装结构和粉丝确认进行调整。

## 当前状态

本仓库提供详细 SOP 和未来自动化接口设计，不代表已经存在“一条命令生成任意 idol”的完整工具。现阶段仍需要：

1. 合法获得或公开可见的参考素材。
2. 身份锚点和主视觉的人工确认。
3. 分行动作生成与视觉审核。
4. 私有素材工作区和构建环境。
5. 干净 Windows 环境烟测与发布授权。

## 推荐使用方式

1. 先选择女性版或男性版作为动作起点。
2. 填写规范中的 `idol-project.yaml` 项目清单。
3. 先批准正面全身主视觉，再批量生成动作。
4. 按联系表逐行动作审核，失败时只修最小范围。
5. 完成自动 QA、实际桌面预览和权利说明后再公开发布。
