# 偶像 Windows 互动角色开发规范

本仓库为偶像主题的 Windows 互动角色提供可执行的开发规范、机器契约、
QA 门禁和发布边界。目标是让不同粉丝项目在保持人物特色的同时，拥有一致、
可验证的交互体验与安全标准。所有规范仅用于免费、非官方、非商业项目。

本仓库的两份规范是写给 AI 开发者执行的，不是灵感参考，也不是允许 AI 自选技术方案的产品建议。目标只有一个：**以“小曦薇 v3.0.6”的已验证成品为固定底座，更换人物与动作视觉，并强制应用本规范新增的“全部官方皮肤内置单 EXE”和“当前用户级开机自启”扩展；其他能力、协议、交互、QA 和安全边界保持一致。**

## 给 AI 的最高优先级指令

当用户要求“按本规范开发某位 idol”时，必须遵守以下顺序：

1. 完整读取所选的女性版或男性版规范，以及机器可读契约。
2. 从 [`super-idol-hub/xiaoxiwei` 的 `v3.0.6`](https://github.com/super-idol-hub/xiaoxiwei/tree/v3.0.6) 开始，基线提交固定为 `e9b77f9bb106c8e83c3977e853b7f9eba9e6c1bc`。
3. 保留模板提交历史进行 clone 或 fork，再只做规范白名单内的人物、动作、产品身份和公开动态来源替换。
4. 不得从空项目重新设计，不得更换技术栈，不得删减“不好做”的功能。
5. 生成一个包含全部官方皮肤的 Windows EXE、完整 ZIP、完整 QA 证据和 SHA-256 后，才能声称完成。
6. 右键菜单必须提供“开机自启”；全部官方皮肤必须在同一个 EXE 的“皮肤”菜单中切换，不能拆成多个 EXE，也不能要求用户携带官方 `skins` 文件夹。

若用户的新要求与冻结协议冲突，AI 必须停止该冲突项并报告，不得自行选择替代框架、降低帧数、删除动作或交付概念稿。

**基线差异门禁**：开工时必须证明 `git rev-parse HEAD` 为上述固定提交。交付时必须附上相对该提交的 `git diff --name-status`、`git diff --stat` 和 `baseline-proof.json`，把每一个变化文件映射到允许变化域。身份替换登记到 `identity-replacements.json`；本规范批准的两个运行时扩展登记到 `protocol-extensions.json`。除此之外，复制后重写运行时、脚本或协议，即使功能名相似，也视为没有使用小曦薇基线。

每个变化路径必须按机器契约恰好命中一个白名单规则；每项源码字符串／标识符替换必须登记在 `identity-replacements.json`，反向替换后与基线 blob 的 SHA-256 完全相同。内部实现目录固定保留为 `source/standalone/xiaoxiwei/`，新 idol 的 slug 只用于产品身份与发行物，不用目录改名制造一套新运行时。

## 不可变核心

- C# 5、经典 .NET Framework、WinForms 分层透明窗口。
- 全部官方皮肤及其运动网格嵌入同一个单文件 `winexe`；运行时不依赖 Codex、Python、OpenCV、浏览器、同目录应用 DLL 或官方 `skins` 文件夹。
- 24 行、176 张 `528×808` RGBA 关键帧；逻辑渲染尺寸 `132×202`。
- 运行时至少 24 个显示阶段，使用 `17×25` 双向运动网格和单一人物轮廓形变，不做两张完整人物交叉淡化。
- r09/r10 的 16 方向近距离看鼠标；离开范围后回到呼吸眨眼。
- 拖拽受惊、松手跺脚生气；不启用走路、慢跑、快跑或自动漫游。
- 持续玩手机、持续睡眠与头顶 `z/Z`、单击退出／唤醒。
- 皮肤 API v1 的 ID、元数据、176 帧和运动网格契约保持不变；它用于构建期官方皮肤归档和可选第三方兼容皮肤。
- 右键“皮肤”菜单必须列出全部官方内置皮肤，切换选择持久化，损坏候选不得破坏当前皮肤。
- 右键公开动态功能下方必须提供“开机自启”；使用当前用户启动项，不要求管理员权限，关闭后删除本产品自己的启动项。
- 人物 alpha 切片伪 3D 转身、由头到脚变装、r21/c07 定格、回待机；禁止整张卡片翻转。
- `--self-test`、`--qa-window`、帧／运动／皮肤／交互／隐私检查、干净 Windows 烟测和 SHA-256。
- 开发者固定为 `Anbunensi`，Company 留空；非官方、非商业免责声明必须保留。
- 成品《使用说明》只面向普通用户介绍启动、互动、皮肤菜单切换、开机自启和退出，不得出现皮肤接口、Skin API、`skin.xml`、`frames.zip`、外置 `skins` 安装或开发者接入教程。
- 本仓库的机器契约、动作帧数／时长数组、状态机常量和验收脚本本身属于冻结内容，不能先改契约再宣布新实现“合规”。

## 默认禁止

除非项目负责人另行给出带编号的协议变更批准，AI 不得：

- 改成 Electron、网页、Python、Unity、Godot、Live2D、WPF 或 .NET 6+；
- 把通用原型工具生成的 `8×11` 小图集当作最终 Windows 成品；
- 改动作行号、归档帧数、关键帧时长、状态机优先级、皮肤路径或 `.mtn` 格式；
- 重新启用移动行、自动漫游或拖拽走跑；
- 用 GIF、视频、静态贴图、整图透明淡化或整窗翻牌代替既定运行时；
- 因动作难做而删除手机、睡眠、看鼠标、拖拽反应、换肤或托盘功能；
- 把不同官方皮肤拆成多个 EXE，或要求用户携带官方 `skins` 文件夹；
- 缺少“开机自启”菜单、写入全局／管理员级启动项，或在自检模式中修改注册表；
- 只交设计图、伪代码、SOP、演示视频或“未来可完成”的工程骨架；
- 把参考原图、Cookie、Token、账号凭据、节目音视频或私有绝对路径提交到公开仓库。

## 正确使用方式

1. 选定女性版或男性版。
2. 把该文档完整提供给执行 AI，并明确 idol、主造型、附加皮肤和是否公开发布。
3. 要求 AI 先输出“基线确认 + 白名单变更表”，确认它确实以 `v3.0.6` 为底座。
4. 先批准正面全身身份锚点，再批准逐行动作联系表。
5. 要求 AI 实际构建并运行 `--self-test` 与 `--qa-window`，不得用文字声明代替结果。
6. 最后验收单一 EXE 内的全部官方皮肤、开机自启开关、完整 ZIP、QA 目录、哈希和干净环境烟测。
