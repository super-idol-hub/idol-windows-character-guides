# Windows 互动角色开发规范

这是一份可直接交给 AI 执行的 Windows 互动角色开发规范。它覆盖人物素材整理、角色视觉定稿、动作资源、交互逻辑、皮肤切换、Windows 构建、自动化测试和正式发布。

目标不是生成一个只能演示的窗口，而是一次完成一款可长期使用和分发的正式角色：

- 人物形象稳定，动作与本人特点相符；
- 一个 EXE 内置全部官方角色皮肤和动作资源；
- 支持拖动、凝视、挥手、手机、睡眠等完整互动；
- 在干净的 Windows 环境中无需安装 Python、OpenCV 或额外运行库；
- 有自动测试、人工验收记录、校验值和清晰的发布包；
- 不依赖旧项目的目录、提交号或历史版本迁移。

## 给 AI 的使用方式

把本仓库的主规范和人物资料一起提供给 AI，并使用下面的任务说明：

```text
请完整阅读《Windows 互动角色开发规范》，根据我提供的人物资料开发一款正式的
Windows 互动角色。以规范中的“完成定义”为唯一交付标准，不要只制作原型，
不要生成多个过渡版本，也不要在没有运行构建和测试的情况下声称完成。

角色名称：
项目标识：
官方皮肤：
人物资料目录：
目标仓库：
发布授权或使用边界：
```

AI 应先确认最少输入，再按“视觉定稿 → 资源生产 → 程序实现 → 自动测试 → 人工验收 → 正式发布”推进。除人物身份、皮肤方案和授权边界外，不应把规范已经确定的技术问题反复交给用户选择。

## 主规范

[阅读《Windows 互动角色开发规范》](docs/Windows互动角色开发规范.md)

主规范包含：

- 一次性交付流程与项目目录；
- 24 行、176 帧动作协议；
- 无边框透明窗口、托盘、缩放、拖动、凝视和持久动作；
- 内置皮肤、动作网格和资源校验；
- C# / .NET Framework 构建要求；
- 自动化 QA、人工验收、发布包和权利边界；
- AI 的停止条件、失败处理和正式完成定义。

## 参考实现

[xiaoxiwei](https://github.com/super-idol-hub/xiaoxiwei) 是本规范所描述产品形态的参考实现。参考它的最终体验和工程思路即可，不要求继承某个历史提交、旧目录或旧版本差异。

新角色应使用自己的中性项目标识和目录，例如：

```text
source/standalone/<character-slug>/
```

不得把其他人物的名称、资源、注册表键、程序集信息或发布说明残留在新项目中。

## 适用边界

本规范用于获得授权的互动角色项目。默认定位为非官方、非商业的粉丝向软件；如项目已取得其他授权，以实际书面授权为准。不得将未获许可的照片、视频、音频、歌词、登录凭据或平台 Cookie 放入源码和发布包。

开发者署名统一使用 **Anbunensi**；项目可单独列出素材来源、贡献者与权利说明。
