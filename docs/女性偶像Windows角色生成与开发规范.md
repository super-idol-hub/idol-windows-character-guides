# 女性偶像 Windows 互动角色生成与开发规范

> 面向 AI 开发者的强制执行契约。目标是复制“小曦薇 v3.0.6”的完整开发模式，只替换人物身份、造型、动作视觉、角色文案和该人物的公开动态来源。没有明确列入白名单的内容一律不得改变。

执行本规范的 AI 必须先确认：

```text
我将从 super-idol-hub/xiaoxiwei 的 v3.0.6 开始，
固定基线提交 e9b77f9bb106c8e83c3977e853b7f9eba9e6c1bc。

我只修改本规范白名单内的人物、造型、动作视觉、角色文案、项目身份字符串和公开动态来源。
我会交付实际可运行、包含全部官方皮肤的单一 EXE、完整 ZIP、QA 证据和 SHA-256，
并提供当前 Windows 用户可随时开关的“开机自启”，
而不是概念图、伪代码、SOP 或未完成的工程骨架。
```

必须完整读取本文件和 [`../contracts/xiaoxiwei-v3.0.6.contract.json`](../contracts/xiaoxiwei-v3.0.6.contract.json)。契约定义不可变行为和验收边界，固定提交定义唯一实现基线。两者出现冲突、缺项或无法验证时，必须停止并报告 `contract-source-conflict`；不得以“源码优先”为理由扩大白名单、改变协议或删减验收。

本规范中的“必须”“不得”“仅允许”是发布阻断条件，不是建议。出现冲突时，AI 只能报告冲突并等待项目负责人批准带编号的例外，不得自行选择“更现代”“更简单”或“差不多”的实现。

## 1. 权威基线与复制方式

唯一权威模板：

- 仓库：`https://github.com/super-idol-hub/xiaoxiwei`
- Tag：`v3.0.6`
- Commit：`e9b77f9bb106c8e83c3977e853b7f9eba9e6c1bc`
- 产品形态：C# WinForms 分层透明窗口、一个 Windows EXE 内置全部官方皮肤；同级 `skins` 仅保留为可选第三方兼容接口

本规范在固定基线上批准且强制应用两个组织级扩展：

- `GUIDE-EXT-001-EMBED-ALL-OFFICIAL-SKINS`：全部官方皮肤及其完整运动网格作为程序集资源嵌入同一个 EXE，由“皮肤”菜单切换。
- `GUIDE-EXT-002-CURRENT-USER-AUTOSTART`：右键菜单增加“开机自启”，只管理当前 Windows 用户、本产品自己的启动项。

这两个扩展必须登记在 `protocol-extensions.json`；它们不是允许任意重写运行时的通行证。除扩展所需的资源目录、菜单、注册表和构建／打包连接点外，固定基线仍按默认禁止执行。

新项目必须从该提交 clone 或 fork，并在 Git 历史中保留该基线。AI 不得只阅读本规范后凭记忆重新实现，不得复制后创建无关联的新历史，也不得把其他互动角色框架包装成“兼容版”。开始编码前必须保存：

1. 模板提交号；
2. 新项目显示名、ASCII slug、EXE 名、仓库名；
3. 白名单变更表；
4. 明确声明未改动的协议项；
5. 人物素材的私有来源登记位置。

基线差异门禁：

```powershell
git rev-parse HEAD
git show --no-patch --format=%H v3.0.6
git diff --name-status e9b77f9bb106c8e83c3977e853b7f9eba9e6c1bc..HEAD
git diff --stat e9b77f9bb106c8e83c3977e853b7f9eba9e6c1bc..HEAD
```

开工时前两条必须得到固定提交；交付时后两条和 `baseline-proof.json` 必须把每个变化文件映射到白名单。路径按机器契约的大小写敏感、仓库相对完整 glob 规则匹配；每个变化路径必须恰好命中一条规则，零命中或多命中都失败。`XiaoXiWeiPet.cs` 除身份／文案替换外，只可为 `GUIDE-EXT-001` 增加官方内置皮肤资源目录、加载与菜单连接点，为 `GUIDE-EXT-002` 增加当前用户启动项菜单和幂等读写；不得改变动作协议、插值算法、状态枚举和输入优先级。构建脚本只可增加“逐套验证归档 → 全部资源链接进同一 EXE → 单 EXE 打包”的连接点。复制后重写或以“等价实现”替换冻结文件，直接失败。

身份、路径和产品文案替换必须写入 `identity-replacements.json`；两个组织级扩展的每个新增或修改 hunk 必须写入 `protocol-extensions.json`，记录扩展 ID、文件、符号、目的和对应 QA。反向应用身份替换并排除已登记扩展 hunk 后，剩余内容必须与基线 blob 完全一致。任何未声明 diff hunk、无关控制流／算法改动、动作协议数字改动或冻结文件哈希不一致都直接失败。人物关键帧、`.mtn` 和 QA 图属于生成产物或忽略的工作素材，不能作为扩大运行时代码改动的理由。

通用原型工具生成的 `8×11`、`192×208` 图集可以用于早期人物或方向实验，但绝对不能作为最终 Windows 互动角色、不能直接放大为 4K 帧，也不能替代本规范的 24 行运行时。

## 3. 冻结的技术实现

### 3.1 编译与运行时

- 编译器：`C:\Windows\Microsoft.NET\Framework64\v4.0.30319\csc.exe`
- 固定参数：`/target:winexe /optimize+ /debug- /platform:anycpu /langversion:5 /codepage:65001`
- 固定引用：`System.dll`、`System.Core.dll`、`System.Drawing.dll`、`System.IO.Compression.dll`、`System.Windows.Forms.dll`
- 入口：`[STAThread]`
- Manifest：`asInvoker`、`uiAccess=false`、Common Controls v6、PerMonitorV2/PerMonitor DPI aware
- 每套官方皮肤的 176 帧和完整运动网格归档均作为独立程序集资源嵌入同一个 EXE；图标和 manifest 一并编译
- 官方皮肤不得拆分成多个 EXE，也不得要求同级官方 `skins` 文件夹
- 正常模式单实例；`--qa-window` 使用独立 QA Mutex
- 命令行：`--self-test [report.json] [preview.png]`、`--qa-window [milliseconds]`

Python、NumPy、Pillow 和 OpenCV 只允许出现在构建期；发布 EXE 运行时不得依赖它们，也不得依赖 Codex 或同目录应用 DLL。

### 3.2 窗口与桌面行为

- 无边框、不进入任务栏、默认置顶、手动定位、右下角启动；
- `WS_EX_LAYERED | WS_EX_TOOLWINDOW`，只显示透明人物；
- 默认缩放 `225%`，范围 `125%–400%`，滚轮每档 `25%`；
- 右键菜单保留动作、皮肤、公开动态提醒、置顶、暂停、帮助和退出；
- “立即检查公开动态”下方增加“开机自启”；首次正常运行默认开启，用户取消勾选后必须删除本产品自己的当前用户启动项并记住关闭状态；
- 开机自启固定使用 `HKCU\Software\Microsoft\Windows\CurrentVersion\Run`，命令为带引号的当前 `Application.ExecutablePath`；不得写 `HKLM`、计划任务、服务或需要管理员权限的位置；
- EXE 移动后再次启用或在启用状态正常启动时必须幂等更新路径；写入失败不得中断角色运行，自检与 QA 模式不得修改注册表；
- 托盘图标保留，托盘双击恢复并移回右下角；
- `Ctrl+Q` 退出；
- 行为定时器和动作状态机沿用模板，不得改成另一个事件模型。

如果新项目需要替换显示名，只做系统性的身份字符串替换，同时保留单实例、日志、托盘和 QA 行为。

## 5. 不可变交互状态机

### 5.1 拖拽、单击与双击

- 拖拽阈值：`max(8px, round(4 × scale))`；默认 `225%` 时为 9px；
- 超过阈值：播放 r04/c00–c02，拖动期间保持 c02；
- 松手：从 r11/c02 开始播放 4 帧，共 1700ms；
- 普通单击不得触发生气；
- 双击必须触发 r03 挥手；
- 鼠标捕获意外丢失不得触发生气；
- 不得重新启用走路、慢跑、快跑和自动漫游。

左键按下的所有权顺序冻结为：正在换肤 → 睡眠／唤醒 → 手机／退出 → 临安持续秋千／退出 → 普通拖拽候选。前四种状态必须消费单击并抑制双击，不能继续落入拖拽判定；只有前四种状态均未持有输入时才能开始普通拖拽候选。

### 5.2 近距离看鼠标

- 进入外半径 220 逻辑像素，退出外半径 270；
- 进入死区 45，退出死区 34；
- 扇区保持角 14.75°，16 个方向每格 22.5°；
- 只在普通 Idle、无临时动作、无拖拽、无换肤时启用；
- 鼠标太近或太远时立即退出 r09/r10 并回到 r00；其他动作开始时停止显示 r09/r10，直接播放目标动作，结束后按状态机回 r00；
- 离开范围后普通待机只呼吸眨眼，不得持续盯着鼠标或左右看；
- r09/r10 必须复用 r00 的人物大小、脚底和下半身中心，不能看鼠标时缩小。

### 5.3 持续动作

- r22：`0→1→2→3` 进入；`3→4→5→4` 循环；单击后 `6→7` 退出；
- r23：`0→1→2→3→4` 入睡；c04 保持并从头顶上方飘 `z/Z`；单击后 `5→6→7` 醒来；
- 手机、睡眠的单击必须被各自状态消费，不能同时变成拖拽或双击第一下；
- 未获编号例外时，新 idol 的 r15 固定为一次性 8 帧动作。只有模板中 `id="linan-princess"` 的既有 r15 使用 `0–2` 进入、`2–5` 循环、`6–7` 点击退出；不得仅凭动作名称让其他皮肤持续。迁移该特例必须把 `LinanSwingContract.SkinId` 映射列为明确例外并同步机器契约。

## 6. 动画平滑与无重影要求

- 每套皮肤的构建归档只保存 176 张关键帧，不保存运行时补间 PNG；
- 每个普通循环的显示阶段数为 `关键帧数 × ceil(24 / 关键帧数)`：8 帧→24、7 帧→28、6 帧→24、5 帧→25、4 帧→24；
- `7→28` 只是通用计算结果；r00 的运行循环固定为 6 帧→24 阶段，c06 不参与循环；
- 动作总节奏按 v3.0.6 的时长数组和旧 56 阶段参考节奏保持；不得因为阶段数变少而整体加速；
- 使用 smoothstep 与 `17×25`、逻辑 `132×202` 的双向 `.mtn` 运动网格；
- 每个中间阶段只能有一个不透明人物 owner，前半段以 from 为 owner，后半段以 to 为 owner；
- 禁止同时绘制两张完整人物做 alpha 交叉淡化；
- 不能逐帧按自身边界放大，必须按整行动作统一视口、统一缩放、统一脚底／主体注册；
- 全部官方内置皮肤、可选第三方皮肤和正式 Release 必须包含完整运动网格。兼容回退不能被当作正式发布方案。

## 7. 皮肤 API v1、官方内置皮肤与换肤

皮肤 API v1 仍作为每套皮肤的构建期元数据和可选第三方兼容格式：

```text
构建输入/
  <skin-id>/
    skin.xml
    frames.zip
```

固定清单：

```xml
<skin apiVersion="1"
      id="skin-id"
      name="皮肤显示名"
      developer="Anbunensi"
      exclusiveAction="可选专属动作名"
      archive="frames.zip" />
```

强制规则：

- 发布清单中声明的每一套官方皮肤都必须嵌入同一个 EXE，并在右键“皮肤”菜单中可见；菜单切换不得启动第二个进程或第二个 EXE；
- 官方皮肤不得依赖 EXE 同级 `skins` 文件夹。该目录只允许用户自行添加兼容的第三方皮肤，不能用于规避官方内置要求；
- 文件夹名等于 `id`；ID 匹配 `^[A-Za-z0-9][A-Za-z0-9._-]{0,63}$`；
- `apiVersion="1"`、`archive="frames.zip"` 固定；文本字段最长 64 字符；
- `frames.zip` 根目录直接是 `r00/c00.png`，不能再套 `frames/`；
- 同一个 `frames.zip` 还必须包含 `motion/rNN/cNN-rNN-cNN.mtn`；普通官方皮肤恰好 313 个 `XWM1` 网格，临安持续秋千恰好 315 个，并额外包含 `motion/r15/c05-r15-c02.mtn` 与 `motion/r15/c02-r15-c06.mtn`；
- 单条目不超过 32 MiB，ZIP 与解压后总量均不超过 128 MiB；
- 冻结运行时会拒绝缺失／重复 PNG、非帧非 motion 文件、不安全路径、尺寸错误、超限条目，以及长度不等于编码长度或前四字节不是 `XWM1` 的 motion；它不会拒绝重复 motion 或名称合法但不在预期集合中的 motion，完整头字段只在请求该网格渲染时解析；
- 正式官方皮肤必须同时具备 `skin.xml`、176 PNG 和完整运动网格，经过独立发布 QA 后再嵌入 EXE；实际有向帧对集合必须与机器契约精确相等，并验证完整长度、magic、17×25 网格和 132×202 逻辑头；
- 全部官方内置皮肤的 r12–r14 行为按各皮肤目录声明执行，r15 为该皮肤专属动作；同一动作在不同皮肤中不得出现人物尺寸突变；
- 可选第三方皮肤继续遵守 API v1 外置可达性和安全上限，但第三方兼容不计入官方交付皮肤数量；
- 候选皮肤先完整校验和预热，结束时原子切换；损坏时回退内置皮肤；
- 连续选择只保留最后一次，不能造成无界缓存。

换肤固定为 16 个人物 alpha 水平切片：14 阶段旋身、24 阶段由头到脚揭示、8 阶段定格、14 阶段回稳，共 60 阶段、约 1200ms。结束姿势固定取 r21/c07，再回 r00。禁止旋转整张矩形窗口或做卡片翻面。

## 8. 人物与动作素材流水线

### 阶段 A：私有参考登记

至少准备：

- 2–4 张同一时期的清晰正面／近正面脸；
- 左右三分之四角度各 1–2 张；
- 自然笑、闭眼笑、严肃、委屈／生气等表情；
- 至少 1 张能看清腿、鞋和真实比例的正面全身；
- 同套造型半身、服装、发型、首饰和鞋的细节图。

原图、来源清单、Cookie、Token 和绝对用户路径只能留在私有工作区并加入 `.gitignore`。公开仓库只保留文字说明，不上传原图或节目视频。

### 阶段 B：身份锚点

先生成一张正面全身中性姿势，锁定：脸型、眼距、眼型、鼻唇关系、下颌、发色、刘海、发长、真实头身比、肩腰腿比例、服装结构、鞋和常驻配饰。

在用户明确批准这张身份锚点前，AI 不得批量制作 176 帧。脸部参考负责身份，全身参考负责比例，造型参考负责服装细节，三者不能互相替代。

### 阶段 C：动作圣经与关键姿势

动作不得只根据“女性”二字推断。先记录该人物的站姿、惯用手、眼神交流、动作能量、手势开合、表情幅度，以及批准造型实际存在的头发、衣摆、裙摆、裤脚、外套、袖口、配饰和常驻道具约束，再记录 2–3 个签名动作。不存在的部件不得因性别自动添加。

每行先做完整关键姿势联系表并人工批准，再进入透明化、统一注册、运动网格和运行时预览。一个动作不自然时只重做该行；发色漂移只修该动作；看鼠标缩放只修 r09/r10。不得通过重写运行时掩盖错误素材。

### 阶段 D：最终 4K 关键帧

- 每张 `528×808` RGBA；
- 人物完整、无背景、无阴影、无水印、无字幕、无品牌大字；
- 同一动作整行共享尺度、视口和注册；
- 不允许四肢、头发、衣摆、裙摆、裤脚、外套、饰物或手机断裂；
- 不允许为动作临时换发色、服装、首饰、年龄或体型；
- 不允许独立漂浮爱心、动作线、烟尘、音符或 UI。

## 9. 必须复用的构建流水线

不得把脚本名摊平成一条虚假的全局顺序。模板的帧和运动构建顺序保持不变，并增加统一的官方皮肤嵌入阶段：

## 10. AI 必须提交的 QA 证据

### 10.1 自动门禁

- 归档恰好 24 行、176 张 `528×808` RGBA；
- 每行按归档数提供真正不同且语义完整的关键姿势；行计数与机器契约完全一致；透明区隐藏 RGB、连通区、内部透明洞、边缘裁切、共享尺度、脚底／主体注册、身份与颜色漂移检查全部通过；
- `.mtn` 完整、`motionFieldLoaded=true`、`ghostFreeSinglePoseOwner=true`；
- 普通循环至少 24 阶段，时长保持；
- r09/r10 直接渲染、人物尺寸与 r00 一致；
- 移动与自动漫游为 false；
- 拖拽、手机、睡眠、r15 专属动作和换肤契约通过；
- 皮肤安全上限、路径、重复条目和原子切换通过；
- EXE 自检报告嵌入皮肤数量、ID、资源名和逐套解码结果；发布清单中的每套官方皮肤均可在同一进程中切换，删除同级 `skins` 后仍全部可用；
- 开机自启开／关／再次开启幂等测试通过；只读写当前用户、本产品自己的值，路径带引号且能随 EXE 位置更新；`--self-test` 与 `--qa-window` 不修改注册表；
- EXE `--self-test` 输出 `ok=true`、`errors=[]`；
- `--qa-window` 实际启动并正常自动退出；
- 发布 EXE 再次自检；生成 SHA-256。

### 10.2 人工视觉门禁

必须在 125%、225%、400% 和深浅背景下检查：

1. 第一眼能否辨认人物；
2. 176 帧是否始终同脸、同发色、同年龄、同体型、同服装；
3. 动作是否无尺寸突跳、脚底跳、无意横移和双人重影；
4. r09/r10 四个基数方向是否明确，远处鼠标是否回普通眨眼；
5. r22/r23 是否持续、单击退出是否完整；
6. `z/Z` 是否从当前睡姿头部上方产生；
7. 换肤是否只旋转人物轮廓、由头到脚更新、最终 pose 正确；
8. 离线、损坏第三方皮肤和连续换肤是否不破坏主程序；移除同级 `skins` 后全部官方皮肤是否仍可切换；
9. 是否保持头身比、肩宽、胸腰比例、肌肉量、腿长、年龄和体型；女性胸腰夸张只是常见失败示例，不代表其他漂移可以忽略。
10. 开机自启菜单位置、勾选状态、关闭后不再启动和重新开启后的当前 EXE 路径是否正确。

任何失败只能修对应动作行、对应皮肤或对应身份字符串。不得以删除功能、降低帧数、关闭检查、声明“可选”或换技术栈来通过。
