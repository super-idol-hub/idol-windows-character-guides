## 3. 冻结的技术实现

### 3.1 编译与运行时

- 编译器：`C:\Windows\Microsoft.NET\Framework64\v4.0.30319\csc.exe`
- 固定参数：`/target:winexe /optimize+ /debug- /platform:anycpu /langversion:5 /codepage:65001`
- 固定引用：`System.dll`、`System.Core.dll`、`System.Drawing.dll`、`System.IO.Compression.dll`、`System.Windows.Forms.dll`
- 入口：`[STAThread]`
- Manifest：`asInvoker`、`uiAccess=false`、Common Controls v6、PerMonitorV2/PerMonitor DPI aware
- 内置 `frames-4k.zip` 作为程序集资源嵌入 EXE；图标和 manifest 一并编译
- 正常模式单实例；`--qa-window` 使用独立 QA Mutex
- 命令行：`--self-test [report.json] [preview.png]`、`--qa-window [milliseconds]`

Python、NumPy、Pillow 和 OpenCV 只允许出现在构建期；发布 EXE 运行时不得依赖它们，也不得依赖 Codex 或同目录应用 DLL。

### 3.2 窗口与桌面行为

- 无边框、不进入任务栏、默认置顶、手动定位、右下角启动；
- `WS_EX_LAYERED | WS_EX_TOOLWINDOW`，只显示透明人物；
- 默认缩放 `225%`，范围 `125%–400%`，滚轮每档 `25%`；
- 右键菜单保留动作、皮肤、微博提醒、置顶、暂停、帮助和退出；
- 托盘图标保留，托盘双击恢复并移回右下角；
- `Ctrl+Q` 退出；
- 行为定时器和动作状态机沿用模板，不得改成另一个事件模型。

如果新项目需要替换显示名，只做系统性的身份字符串替换，同时保留单实例、日志、托盘和 QA 行为。

## 5. 不可变交互状态机

### 5.1 拖拽、单击与双击

- 拖拽阈值：`max(8px, round(4 × scale))`；默认 `225%` 时为 9px；
- 超过阈值：播放 r04/c00–c02，拖动期间保持 c02；
- 松手：从 r11/c02 开始播放 4 帧，共 1700ms；
- 普通单击不得触发生气；
- 双击必须触发 r03 挥手；
- 鼠标捕获意外丢失不得触发生气；
- 不得重新启用走路、慢跑、快跑和自动漫游。

左键按下的所有权顺序冻结为：正在换肤 → 睡眠／唤醒 → 手机／退出 → 临安持续秋千／退出 → 普通拖拽候选。前四种状态必须消费单击并抑制双击，不能继续落入拖拽判定；只有前四种状态均未持有输入时才能开始普通拖拽候选。

### 5.2 近距离看鼠标

- 进入外半径 220 逻辑像素，退出外半径 270；
- 进入死区 45，退出死区 34；
- 扇区保持角 14.75°，16 个方向每格 22.5°；
- 只在普通 Idle、无临时动作、无拖拽、无换肤时启用；
- 鼠标太近或太远时立即退出 r09/r10 并回到 r00；其他动作开始时停止显示 r09/r10，直接播放目标动作，结束后按状态机回 r00；
- 离开范围后普通待机只呼吸眨眼，不得持续盯着鼠标或左右看；
- r09/r10 必须复用 r00 的人物大小、脚底和下半身中心，不能看鼠标时缩小。

### 5.3 持续动作

- r22：`0→1→2→3` 进入；`3→4→5→4` 循环；单击后 `6→7` 退出；
- r23：`0→1→2→3→4` 入睡；c04 保持并从头顶上方飘 `z/Z`；单击后 `5→6→7` 醒来；
- 手机、睡眠的单击必须被各自状态消费，不能同时变成拖拽或双击第一下；
- 未获编号例外时，新 idol 的 r15 固定为一次性 8 帧动作。只有模板中 `id="linan-princess"` 的既有 r15 使用 `0–2` 进入、`2–5` 循环、`6–7` 点击退出；不得仅凭动作名称让其他皮肤持续。迁移该特例必须把 `LinanSwingContract.SkinId` 映射列为明确例外并同步机器契约。

## 6. 动画平滑与无重影要求

- 皮肤包只保存 176 张关键帧，不保存运行时补间 PNG；
- 每个普通循环的显示阶段数为 `关键帧数 × ceil(24 / 关键帧数)`：8 帧→24、7 帧→28、6 帧→24、5 帧→25、4 帧→24；
- `7→28` 只是通用计算结果；r00 的运行循环固定为 6 帧→24 阶段，c06 不参与循环；
- 动作总节奏按 v3.0.6 的时长数组和旧 56 阶段参考节奏保持；不得因为阶段数变少而整体加速；
- 使用 smoothstep 与 `17×25`、逻辑 `132×202` 的双向 `.mtn` 运动网格；
- 每个中间阶段只能有一个不透明人物 owner，前半段以 from 为 owner，后半段以 to 为 owner；
- 禁止同时绘制两张完整人物做 alpha 交叉淡化；
- 不能逐帧按自身边界放大，必须按整行动作统一视口、统一缩放、统一脚底／主体注册；
- 官方内置皮肤、官方附带皮肤和正式 Release 必须包含完整官方运动网格。兼容回退不能被当作正式发布方案。

## 7. 皮肤 API v1 与换肤

固定目录：

```text
<角色名>.exe
skins/
  <skin-id>/
    skin.xml
    frames.zip
```

固定清单：

```xml
<skin apiVersion="1"
      id="skin-id"
      name="皮肤显示名"
      developer="Anbunengsi"
      exclusiveAction="可选专属动作名"
      archive="frames.zip" />
```

强制规则：

- 文件夹名等于 `id`；ID 匹配 `^[A-Za-z0-9][A-Za-z0-9._-]{0,63}$`；
- `apiVersion="1"`、`archive="frames.zip"` 固定；文本字段最长 64 字符；
- `frames.zip` 根目录直接是 `r00/c00.png`，不能再套 `frames/`；
- 同一个 `frames.zip` 还必须包含 `motion/rNN/cNN-rNN-cNN.mtn`；普通官方皮肤恰好 313 个 `XWM1` 网格，临安持续秋千恰好 315 个，并额外包含 `motion/r15/c05-r15-c02.mtn` 与 `motion/r15/c02-r15-c06.mtn`；
- 单条目不超过 32 MiB，ZIP 与解压后总量均不超过 128 MiB；
- 冻结运行时会拒绝缺失／重复 PNG、非帧非 motion 文件、不安全路径、尺寸错误、超限条目，以及长度不等于编码长度或前四字节不是 `XWM1` 的 motion；它不会拒绝重复 motion 或名称合法但不在预期集合中的 motion，完整头字段只在请求该网格渲染时解析；
- 正式官方皮肤必须同时具备 `skin.xml`、176 PNG 和完整运动网格，并使用独立发布 QA 把实际有向帧对集合与机器契约精确比较，同时验证完整长度、magic、17×25 网格和 132×202 逻辑头；缺失、重复、意外或无效网格均不得发布。不得把运行时最低校验冒充正式发布门禁；
- 内置皮肤的 r12–r14 可进入动作菜单和随机待机，r15 为内置专属动作；外置皮肤的 r12–r14 与 r16–r17 不可达，只有 `exclusiveAction` 非空时 r15 才可达。仅绘制这些行不能新增运行功能；
- 候选皮肤先完整校验和预热，结束时原子切换；损坏时回退内置皮肤；
- 连续选择只保留最后一次，不能造成无界缓存。

换肤固定为 16 个人物 alpha 水平切片：14 阶段旋身、24 阶段由头到脚揭示、8 阶段定格、14 阶段回稳，共 60 阶段、约 1200ms。结束姿势固定取 r21/c07，再回 r00。禁止旋转整张矩形窗口或做卡片翻面。

## 8. 人物与动作素材流水线

### 阶段 A：私有参考登记

至少准备：

- 2–4 张同一时期的清晰正面／近正面脸；
- 左右三分之四角度各 1–2 张；
- 自然笑、闭眼笑、严肃、委屈／生气等表情；
- 至少 1 张能看清腿、鞋和真实比例的正面全身；
- 同套造型半身、服装、发型、首饰和鞋的细节图。

原图、来源清单、Cookie、Token 和绝对用户路径只能留在私有工作区并加入 `.gitignore`。公开仓库只保留文字说明，不上传原图或节目视频。

### 阶段 B：身份锚点

先生成一张正面全身中性姿势，锁定：脸型、眼距、眼型、鼻唇关系、下颌、发色、刘海、发长、真实头身比、肩腰腿比例、服装结构、鞋和常驻配饰。

在用户明确批准这张身份锚点前，AI 不得批量制作 176 帧。脸部参考负责身份，全身参考负责比例，造型参考负责服装细节，三者不能互相替代。

### 阶段 C：动作圣经与关键姿势

动作不得只根据“女性”二字推断。先记录该人物的站姿、惯用手、眼神交流、动作能量、手势开合、表情幅度，以及批准造型实际存在的头发、衣摆、裙摆、裤脚、外套、袖口、配饰和常驻道具约束，再记录 2–3 个签名动作。不存在的部件不得因性别自动添加。

每行先做完整关键姿势联系表并人工批准，再进入透明化、统一注册、运动网格和运行时预览。一个动作不自然时只重做该行；发色漂移只修该动作；看鼠标缩放只修 r09/r10。不得通过重写运行时掩盖错误素材。

### 阶段 D：最终 4K 关键帧

- 每张 `528×808` RGBA；
- 人物完整、无背景、无阴影、无水印、无字幕、无品牌大字；
- 同一动作整行共享尺度、视口和注册；
- 不允许四肢、头发、衣摆、裙摆、裤脚、外套、饰物或手机断裂；
- 不允许为动作临时换发色、服装、首饰、年龄或体型；
- 不允许独立漂浮爱心、动作线、烟尘、音符或 UI。

## 9. 必须复用的构建流水线

不得把脚本名摊平成一条虚假的全局顺序。模板有两条冻结的内部流水线：

1. 内置 EXE：运行 `build.ps1`，由它依次调用 `build_hd_frames.py → validate_action_palette.py → build_motion_fields.py → csc.exe`；
2. 每套官方外置皮肤：运行 `build_skin.ps1`，由它依次调用 `build_hd_frames.py → build_motion_fields.py → pack_external_skin.py`。

两类构建完成后运行 `render_runtime_smoothing_qa.py`、EXE 自检和规定的人工／自动 QA；全部通过后才运行 `package_project.ps1`。不得在外部再次调用 `pack_external_skin.py` 造成重复打包，也不得改变任一内部调用关系。

允许在上述脚本中按 `identity-replacements.json` 精确替换 `xiaoxiwei`、`XiaoXiWei`、产品中文名、皮肤列表、版本、发行历史文件名和角色专属源条带名称；不得改变输入输出协议、安全校验、控制流、计算或发布门槛。

## 10. AI 必须提交的 QA 证据

### 10.1 自动门禁

- 归档恰好 24 行、176 张 `528×808` RGBA；
- 每行按归档数提供真正不同且语义完整的关键姿势；行计数与机器契约完全一致；透明区隐藏 RGB、连通区、内部透明洞、边缘裁切、共享尺度、脚底／主体注册、身份与颜色漂移检查全部通过；
- `.mtn` 完整、`motionFieldLoaded=true`、`ghostFreeSinglePoseOwner=true`；
- 普通循环至少 24 阶段，时长保持；
- r09/r10 直接渲染、人物尺寸与 r00 一致；
- 移动与自动漫游为 false；
- 拖拽、手机、睡眠、r15 专属动作和换肤契约通过；
- 皮肤安全上限、路径、重复条目和原子切换通过；
- EXE `--self-test` 输出 `ok=true`、`errors=[]`；
- `--qa-window` 实际启动并正常自动退出；
- 发布 EXE 再次自检；生成 SHA-256。

### 10.2 人工视觉门禁

必须在 125%、225%、400% 和深浅背景下检查：

1. 第一眼能否辨认人物；
2. 176 帧是否始终同脸、同发色、同年龄、同体型、同服装；
3. 动作是否无尺寸突跳、脚底跳、无意横移和双人重影；
4. r09/r10 四个基数方向是否明确，远处鼠标是否回普通眨眼；
5. r22/r23 是否持续、单击退出是否完整；
6. `z/Z` 是否从当前睡姿头部上方产生；
7. 换肤是否只旋转人物轮廓、由头到脚更新、最终 pose 正确；
8. 离线、损坏皮肤和连续换肤是否不破坏主程序；
9. 是否保持头身比、肩宽、胸腰比例、肌肉量、腿长、年龄和体型；女性胸腰夸张只是常见失败示例，不代表其他漂移可以忽略。

任何失败只能修对应动作行、对应皮肤或对应身份字符串。不得以删除功能、降低帧数、关闭检查、声明“可选”或换技术栈来通过。
