## 3. 冻结的技术实现

### 3.1 编译与运行时

- 编译器：`C:\Windows\Microsoft.NET\Framework64\v4.0.30319\csc.exe`
- 参数：`/target:winexe /optimize+ /debug- /platform:anycpu /langversion:5 /codepage:65001`
- 引用：`System.dll`、`System.Core.dll`、`System.Drawing.dll`、`System.IO.Compression.dll`、`System.Windows.Forms.dll`
- 入口：`[STAThread]`
- Manifest：`asInvoker`、`uiAccess=false`、Common Controls v6、PerMonitorV2/PerMonitor DPI aware
- 每套官方皮肤的 176 帧和完整运动网格归档、图标、manifest 均编译进同一个单文件 EXE
- 官方皮肤不得拆分成多个 EXE，也不得要求同级官方 `skins` 文件夹
- 正常模式单实例，QA 模式独立 Mutex
- 命令行：`--self-test [report.json] [preview.png]`、`--qa-window [milliseconds]`

Python、NumPy、Pillow、OpenCV 只用于构建；发布 EXE 不能依赖它们、Codex、浏览器或同目录应用 DLL。

### 3.2 窗口与桌面行为

- 无边框、不进入任务栏、默认置顶、右下角启动；
- 使用 `WS_EX_LAYERED | WS_EX_TOOLWINDOW` 和 `UpdateLayeredWindow`；
- 默认 225%，可在 125%–400% 之间按 25% 滚轮步进；
- 右键菜单保留动作、皮肤、公开动态提醒、置顶、暂停、帮助、退出；
- “立即检查公开动态”下方增加“开机自启”；首次正常运行默认开启，用户取消勾选后必须删除本产品自己的当前用户启动项并记住关闭状态；
- 开机自启固定使用 `HKCU\Software\Microsoft\Windows\CurrentVersion\Run`，命令为带引号的当前 `Application.ExecutablePath`；不得写 `HKLM`、计划任务、服务或需要管理员权限的位置；
- EXE 移动后再次启用或在启用状态正常启动时必须幂等更新路径；写入失败不得中断角色运行，自检与 QA 模式不得修改注册表；
- 托盘图标和托盘双击恢复保留；
- `Ctrl+Q` 退出；
- 产品名替换只允许系统性修改身份字符串，不改变窗口、托盘或输入架构。

## 5. 不可变交互状态机

### 5.1 拖拽、单击与双击

- 阈值：`max(8px, round(4 × scale))`，默认 225% 时为 9px；
- 超阈值后 r04/c00–c02，拖动期间保持 c02；
- 松手后 r11/c02–c05，共 1700ms；
- 普通单击不能生气；双击必须挥手；捕获丢失不能生气；
- 不重新启用走路、慢跑、快跑和自动漫游。

左键按下的所有权顺序冻结为：正在换肤 → 睡眠／唤醒 → 手机／退出 → 临安持续秋千／退出 → 普通拖拽候选。前四种状态必须消费单击并抑制双击，不能继续落入拖拽判定；只有前四种状态均未持有输入时才能开始普通拖拽候选。

### 5.2 近距离看鼠标

- 进入／退出外半径：220／270 逻辑像素；
- 进入／退出死区：45／34；扇区保持角 14.75°；
- 16 方向每格 22.5°；
- 只在普通 Idle 且无临时动作、拖拽或换肤时启用；
- 鼠标太近或太远时立即退出 r09/r10 并回到 r00；其他动作开始时停止显示 r09/r10，直接播放目标动作，结束后按状态机回 r00；
- 离开范围后只呼吸眨眼，不得持续盯着鼠标；
- r09/r10 与 r00 同尺度、同脚底、同下半身中心，不能看鼠标时缩小或变高。

### 5.3 持续动作

- r22：c00–c03 进入，`3→4→5→4` 循环，单击后 c06–c07 退出；
- r23：c00–c04 入睡，c04 保持，头顶 `z/Z`，单击后 c05–c07 唤醒；
- 对持续状态的单击不能同时变成拖拽或双击；
- 未获编号例外时，新 idol 的 r15 固定为一次性 8 帧动作。只有模板中 `id="linan-princess"` 的既有 r15 使用 `0–2` 进入、`2–5` 循环、`6–7` 点击退出；不得仅凭动作名称让其他皮肤持续。迁移该特例必须把 `LinanSwingContract.SkinId` 映射列为明确例外并同步机器契约。

## 6. 动画平滑与无重影要求

- 皮肤包只有 176 张关键帧，补间不额外打包为 PNG；
- 显示阶段数为 `关键帧数 × ceil(24 / 关键帧数)`：8→24、7→28、6→24、5→25、4→24；
- `7→28` 只是通用计算结果；r00 的运行循环固定为 6 帧→24 阶段，c06 不参与循环；
- 关键帧时长、56 阶段参考节奏、最小 16ms Tick 与机器契约一致；
- 使用 smoothstep、`17×25`、逻辑 `132×202` 的双向 `XWM1` 运动网格；
- 一个中间阶段只能有一个不透明人物 owner，禁止两张完整人物透明交叉淡化；
- 整行动作共享视口、缩放、脚底和主体注册；禁止逐帧自动放大导致体型跳变；
- 全部官方内置皮肤、可选第三方皮肤和正式 Release 必须具备完整 `.mtn`；兼容回退不能作为正式交付。

## 7. 皮肤 API v1、官方内置皮肤与换肤

皮肤 API v1 仍作为每套皮肤的构建期元数据和可选第三方兼容格式：

```text
构建输入/
  <skin-id>/
    skin.xml
    frames.zip
```

清单固定：

```xml
<skin apiVersion="1"
      id="skin-id"
      name="皮肤显示名"
      developer="Anbunensi"
      exclusiveAction="可选专属动作名"
      archive="frames.zip" />
```

规则固定：

- 发布清单中声明的每一套官方皮肤都必须嵌入同一个 EXE，并在右键“皮肤”菜单中可见；菜单切换不得启动第二个进程或第二个 EXE；
- 官方皮肤不得依赖 EXE 同级 `skins` 文件夹。该目录只允许用户自行添加兼容的第三方皮肤，不能用于规避官方内置要求；
- 文件夹名等于 ID；ID 匹配 `^[A-Za-z0-9][A-Za-z0-9._-]{0,63}$`；
- `apiVersion="1"`、`archive="frames.zip"` 不变；文本字段最长 64 字符；
- ZIP 根目录直接为 `r00/c00.png`；完整 176 帧；
- 同一个 `frames.zip` 还必须包含 `motion/rNN/cNN-rNN-cNN.mtn`；普通官方皮肤恰好 313 个 `XWM1` 网格，临安持续秋千恰好 315 个，并额外包含 `motion/r15/c05-r15-c02.mtn` 与 `motion/r15/c02-r15-c06.mtn`；
- 单条目 ≤32 MiB，ZIP 与解压后总量 ≤128 MiB；
- 冻结运行时会拒绝缺失／重复 PNG、非帧非 motion 文件、不安全路径、尺寸错误、超限条目，以及长度不等于编码长度或前四字节不是 `XWM1` 的 motion；它不会拒绝重复 motion 或名称合法但不在预期集合中的 motion，完整头字段只在请求该网格渲染时解析；
- 正式官方皮肤必须同时具备 `skin.xml`、176 PNG 和完整运动网格，经过独立发布 QA 后再嵌入 EXE；实际有向帧对集合必须与机器契约精确相等，并验证完整长度、magic、17×25 网格和 132×202 逻辑头；
- 全部官方内置皮肤的 r12–r14 行为按各皮肤目录声明执行，r15 为该皮肤专属动作；同一动作在不同皮肤中不得出现人物尺寸突变；
- 可选第三方皮肤继续遵守 API v1 外置可达性和安全上限，但第三方兼容不计入官方交付皮肤数量；
- 候选皮肤完整校验／预热后原子切换；失败回退内置皮肤；
- 连续选择只保留最后一次。

换肤固定为 16 个人物 alpha 切片，14+24+8+14=60 阶段、20ms/阶段、约 1200ms：人物伪 3D 旋身 → 新造型由头到脚揭示 → r21/c07 定格 → r00。不得旋转整张矩形窗口或做卡片翻面。

## 8. 男性人物素材与动作流水线

### 阶段 A：私有参考登记

至少需要：

- 同一时期 2–4 张清晰正面／近正面脸；
- 左右三分之四角度各 1–2 张；
- 自然笑、开怀笑、严肃、疲惫、委屈或生气等表情；
- 至少 1 张能看清腿、鞋和真实比例的正面全身；
- 同套造型中实际存在的服装、发型、衣摆、裙摆、裤脚、外套、袖口、鞋、配饰和常驻道具细节。

原图和来源登记放私有目录并加入 `.gitignore`。不得把节目视频、歌曲、歌词、台词、字幕、水印、Cookie、Token 或绝对用户路径提交公开仓库。

### 阶段 B：身份锚点

先生成正面全身中性姿势，锁定脸型、眼距、眉眼、鼻唇、下颌、发色、发型、真实头身比、肩宽、腰线、腿长、服装层次、鞋和常驻配饰。

不得把男性体型自动夸大为宽肩、增肌、长腿或超小头。必须以全身参考为准。用户未批准身份锚点前禁止批量制作 176 帧。

### 阶段 C：角色动作圣经

性别不能作为动作参数。记录具体人物的：

- 中性站姿、惯用手、眼神交流方式；
- 动作能量、手势开合程度、节奏快慢、表情幅度；
- 2–3 个签名动作，例如半笑、整理袖口、两拍点头、固定开掌路径；
- 批准造型实际存在的头发、衣摆、裙摆、裤脚、外套、袖口、配饰和常驻道具的物理约束；不存在的部件不得因性别自动添加。

签名动作可以进入 r00、r03、r06–r08、r12–r15、r18–r21，但不能破坏每行固定功能。不得临时给角色增加武器、花、爱心、麦克风或手机之外的道具，除非身份锚点和用户要求明确包含。

### 阶段 D：逐行动作与 4K 帧

每行先做 4–8 个真正不同的关键姿势联系表并人工批准，再做透明化、整行注册、运动网格和实际尺寸预览。

- 每帧 `528×808` RGBA；
- 人物完整、无背景、无阴影、无字幕、无水印；
- 同一行共享缩放、视口、脚底和主体中心；
- 同脸、同发色、同年龄、同身材、同服装、同配饰；
- 无断手、断脚、断发、断衣摆、断裙摆、断裤脚、断外套、道具消失或双重轮廓；
- 无漂浮爱心、动作线、烟尘、音符或 UI；
- 某行失败只重做该行，不能重写运行时掩盖素材问题。

## 9. 必须复用的构建流水线

## 10. AI 必须提交的 QA 证据

### 自动门禁

- 24 行、176 张、每张 `528×808` RGBA；
- 每行按归档数提供真正不同且语义完整的关键姿势；归档／运行计数、时长数组、随机权重与契约一致；
- 透明区隐藏 RGB、连通区、内部透明洞、边缘裁切、共享尺度、脚底／主体注册、身份与颜色漂移检查全部通过；
- `.mtn` 完整、`motionFieldLoaded=true`、`ghostFreeSinglePoseOwner=true`；
- 每个循环至少 24 阶段，节奏未改变；
- r09/r10 直接渲染且与 r00 同尺度；
- 走跑和自动漫游保持关闭；
- 拖拽、手机、睡眠、专属动作、换肤和皮肤安全通过；
- EXE 自检报告嵌入皮肤数量、ID、资源名和逐套解码结果；发布清单中的每套官方皮肤均可在同一进程中切换，删除同级 `skins` 后仍全部可用；
- 开机自启开／关／再次开启幂等测试通过；只读写当前用户、本产品自己的值，路径带引号且能随 EXE 位置更新；`--self-test` 与 `--qa-window` 不修改注册表；
- EXE `--self-test` 为 `ok=true`、`errors=[]`；
- `--qa-window` 实际启动并正常退出；
- 打包后的 Release EXE 再自检；SHA-256 已生成。

### 人工视觉门禁

必须在 125%、225%、400% 与深浅背景检查：

1. 第一眼能否辨认具体人物；
2. 176 帧是否同脸、同年龄、同发色、同体型、同服装；
3. 是否无尺寸突跳、脚底跳、无意横移、断裂和双人重影；
4. r09/r10 四个基数方向是否明确，鼠标离远是否回普通待机；
5. r22/r23 是否持续且单击退出完整；
6. `z/Z` 是否从睡姿头部上方产生；
7. 换肤是否只旋人物、由头到脚更新、最终 pose 正确；
8. 离线、损坏第三方皮肤和快速连续换肤是否安全；移除同级 `skins` 后全部官方皮肤是否仍可切换；
9. 是否保持头身比、肩宽、胸腰比例、肌肉量、腿长、年龄和体型；男性增肌宽肩只是常见失败示例，不代表其他漂移可以忽略；同时不得仅因男性身份删除情绪或把所有动作做成耍帅攻击。
10. 开机自启菜单位置、勾选状态、关闭后不再启动和重新开启后的当前 EXE 路径是否正确。

失败只能修对应动作行、皮肤或身份字符串。不得通过减帧、删功能、关检查、改框架或声明“可选”来过门禁。
